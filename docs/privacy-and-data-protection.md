# Privacy and Data Protection

## Data Collection Summary

| Data Field | Why It Is Needed | Personal Data? | Sensitive Data? | Keep / Remove / Replace | Notes |
|---|---|---|---|---|---|
| Student ID | Identify the student account | Yes | No | Keep | Used only for identification within the system |
| Student Name | Display user information | Yes | No | Keep | Visible only to authorized users |
| University Email | User login and communication | Yes | No | Keep | Used for notifications and account verification |
| Announcement Title | Display campus announcements | No | No | Keep | Public campus information |
| Event Information | Display campus events | No | No | Keep | Shared with all users |
| Project Information | Display student project details | No | No | Keep | Academic project information only |
| Admin Account | Manage announcements and system records | Yes | No | Keep | Accessible only to authorized administrators |

## Privacy Rule for Prototype

The CampusLife Hub prototype collects only the minimum information required to demonstrate the system workflow. Student information such as student ID, name, and university email is used only for identification and communication. Campus announcements, events, and project information may be viewed by users, while administrative functions are restricted to authorized administrators. The prototype does not collect sensitive information such as financial records, health data, biometric information, or real passwords. Mock data is used during usability testing to protect participant privacy.

## Data Minimization Decision

- Real passwords are not stored or displayed in the prototype.
- Personal phone numbers and home addresses are not collected.
- Financial, medical, and biometric information are excluded.
- Only information necessary for the prototype workflow is included.
- Mock user data is used during validation instead of real student records.
