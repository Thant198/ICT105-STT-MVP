# Business Model Canvas

## Project Title

Campus Life – Student Project Management System

## 1. Customer Segments

- University students
- Student project teams
- Club leaders
- Lecturers
- University administrators

## 2. Value Proposition

The Campus Life Management System provides a centralized platform where students can access announcements, manage project tasks, track deadlines, and discover campus events. It reduces missed information, improves team collaboration, and makes campus communication more organized and efficient.

## 3. Channels

- Web application
- University network
- QR code shared in classrooms or events
- Links shared through LINE, Email, or the university LMS

## 4. Customer Relationships

- Easy-to-use interface
- Timely notifications for announcements and deadlines
- Regular updates from administrators
- Feedback collected from students to improve the system

## 5. Revenue Streams / Sustainability Logic

This project is intended as a university support system rather than a commercial product. Its value comes from improving communication, reducing missed deadlines, increasing student engagement, and supporting more efficient campus operations.

## 6. Key Resources

- HTML, CSS, and JavaScript
- GitHub Repository
- Wireframes and UI Prototype
- Project documentation
- Team members
- User feedback collected during Lab 03

## 7. Key Activities

- Develop the web prototype
- Manage announcements, projects, and events
- Maintain project documentation
- Test usability with students
- Update records and improve the system based on feedback

## 8. Key Partners

- Rangsit University
- Lecturers
- Student organizations and clubs
- Project team members
- University IT support (future implementation)

## 9. Cost Structure

- Team development time
- Internet connection
- GitHub (Free)
- Figma / Draw.io (Free)
- Web hosting for the prototype (GitHub Pages)
- Future maintenance and system improvements

## Final Note

The business model directly supports the project's MVP features, including viewing announcements, managing project deadlines, browsing campus events, submitting information, and administrator record management. These features address the problems identified during customer discovery and provide the foundation for the final prototype developed in later labs.
