# Critical Assumptions

## Instruction
Identify assumptions that could cause your final prototype to fail if they are wrong.

| Assumption ID | Category | Assumption | Related Requirement/User Story | Risk Level | Current Evidence | How to Test |
|---|---|---|---|---|---|---|
| A-01 | User problem | Students need one centralized platform for announcements, projects, and events instead of checking multiple communication channels. | FR-01 / US-01 | High | Lab 03 interviews showed students miss important information because it is scattered across LINE, Messenger, Email, and LMS. | Conduct usability testing with the homepage and gather user feedback. |
| A-02 | Value proposition | Students will regularly use the system to track deadlines and announcements. | FR-02 / US-02 | High | Most respondents reported missing deadlines and important updates. | Test prototype usage with students over several tasks and collect feedback. |
| A-03 | Usability | Users can easily navigate the homepage, search records, and locate information without guidance. | FR-05 / US-05 | Medium | Wireframe review only; no usability testing yet. | Perform task-based usability testing and measure completion time. |
| A-04 | Technical feasibility | The system can store and retrieve announcements, events, and project records without performance issues. | FR-03 / US-04 | Medium | Prototype design completed, but no implementation evidence yet. | Develop a simple database prototype and test data retrieval. |
| A-05 | Business logic | Administrators can effectively approve, edit, and update records before they become visible to students. | FR-08, FR-09 / US-06 | Medium | Based on proposed workflow only. | Validate the admin workflow through prototype testing with instructors or staff. |
| A-06 | Data handling | Users will submit complete and accurate information through the submission form. | FR-10, FR-11 / US-04 | Low | Form fields were designed based on user needs, but no real submissions yet. | Test form validation and collect sample user submissions. |

## Categories

Use these categories:
- User problem
- Value proposition
- Usability
- Technical feasibility
- Business logic
- Data handling
