# MVP Experiment Plan

## 1. Group and Project Information
- Group name: [ICT105 STT]
- Project title: Campus Life Hub
- Repository link: https://github.com/Thant198/ICT105-ST-MVP
- Main target user: ICT Students and University Classmates
- Prototype platform: Figma Clickable Wireframe

## 2. Experiment Objective
We want to test whether ICT students can successfully track academic deadlines, access pinned resources, and navigate our layouts without external help.

## 3. Requirement Scope for the Experiment

| Requirement ID | Requirement Summary | Related Screen/Feature | Tested in This Experiment? |
|---|---|---|---|
| FR-01 | View dashboard metrics | Dashboard screen | Yes |
| FR-05 | Browse active task lists | Record list screen | Yes |
| FR-07 | Access task guidelines and links | Detail view screen | Yes |
| FR-09 | Edit/delete task records | Admin view screen | Yes |

## 4. MVP Experiment Type
- Selected experiment type: Clickable prototype

- Reason for selection: 
Using a Figma clickable prototype allows us to rapidly evaluate our core user flows, navigation logic, and layout usability with actual classmates before committing to writing backend code.

## 5. Test Users
| Test User Group | Number of Testers | Why They Are Relevant |
|---|---:|---|
| ICT Students | 5 | They represent our primary target audience who face task management issues daily. |

## 6. Experiment Procedure Summary
The test will be conducted in person or via online screen-sharing. Each tester will be given a specific set of 4 tasks (using our interactive Figma screens) from start to finish. We will observe where they hesitate, measure their task completion rate, and collect feedback using a structured, neutral questionnaire immediately after they finish.

## 7. Expected Learning
We will learn whether our visual navigation and layout design are intuitive. If the success rate is below 70%, we will redesign the navigation hierarchy and button placements before initiating any Frontend HTML/CSS development.
