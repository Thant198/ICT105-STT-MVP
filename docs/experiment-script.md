# Experiment Script

## 1. Opening Script
Thank you for helping us test our prototype. This is not a test of you. We are testing whether our product idea and interface are clear. Please think aloud while using the prototype.

## 2. Test Scenario
You are a student who wants to keep track of your upcoming assignment deadlines, find pinned class resources, and manage academic tasks easily without searching through chaotic Viber/Telegram groups.

## 3. Tasks for Tester
| Task No. | Task Instruction | Related Requirement | Expected Result |
|---|---|---|---|
| T1 | Open the homepage/dashboard and explain what you think this system is for. | FR-01 | User understands the purpose of Campus Life Hub. |
| T2 | Submit or add a new assignment deadline to the list. | FR-03, FR-10, FR-11 | User successfully fills the input form and sees confirmation. |
| T3 | Search or filter the task list specifically for the "ICT105" course. | FR-05, FR-06 | User successfully filters and finds the relevant task. |
| T4 | Open the detail page of the "Lab 07" task. | FR-07 | User views and understands the task details and links. |
| T5 | Access the admin actions to change a task status or delete an outdated record. | FR-08, FR-09 | User understands how to update or delete rows. |

## 4. Observation Notes
Record where the tester hesitates, clicks the wrong place, asks questions, or gives useful feedback during the test.

## 5. Closing Questions
- What was easy to understand?
- What was confusing?
- Which feature (Dashboard vs. Admin Management) was most useful?
- What should be improved before we build the final web prototype?
